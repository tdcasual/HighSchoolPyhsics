# 课堂展示模式设计原则（面向后续新动画）

本文档定义课堂展示模式的基础标准，用于指导后续所有新建演示页面。

## 1. 设计目标

- 让教师在 1080P 大屏上快速进入“可讲状态”。
- 保证核心结论在首屏可见，不依赖复杂操作。
- 允许“自动布局”提升效率，同时支持“手动覆盖”保证稳定性。

## 2. 课堂模式核心原则

1. 有感知
- 开启课堂模式后，界面必须有明确状态反馈（例如“课堂展示中”）。

2. 有保底
- 即使控制区折叠，也要保留核心摘要（`coreSummary`）在首屏可见。

3. 有层级
- 核心信息优先级应为：结论读数 > 当前状态 > 次要参数。

4. 有一致性
- 触屏交互、按钮反馈、折叠行为在各场景保持统一。

## 3. 自动布局与信号策略

### 3.1 信号类型

- `chart`: 图表核心
- `time-series`: 时间序列
- `live-metric`: 实时读数
- `interactive-readout`: 互动观察结果

### 3.2 自动判定建议

- 图表主导场景（示波器/回旋加速器）应默认能进入双核心布局（`split`）。
- 结构观察主导场景（奥斯特/MHD）可默认视口优先（`viewport`），但需保留摘要。

### 3.3 人工兜底

- 顶部布局切换器（自动/双核心/视口优先）始终可覆盖自动判定并按路由记忆。

## 4. 1080P课堂可读性基线

1. 首屏可见
- 课堂模式下核心摘要应在无需滚动的区域出现。

2. 摘要规模
- 建议 3~5 行，每行只放一个核心信息点，避免段落化。

3. 文本选择
- 优先显示“当前状态 + 关键量值 + 结论标签”。

4. 字体策略
- 课堂模式应统一增大标题、读数、交互按钮字号。

## 5. 新场景接入清单（Agent执行）

1. 场景根布局：为 `SceneLayout` 传入 `presentationSignals`。
2. 保底摘要：实现 `coreSummary`，覆盖折叠场景核心信息。
3. 关键节点：给主要信息块加 `data-presentation-signal`。
4. 验证路径：
- 开启课堂模式
- 验证自动布局结果
- 验证折叠后摘要仍在
- 切换到双核心/视口优先，确认覆盖生效
5. 测试要求：
- 至少新增一条“课堂模式保底可见性”测试。

## 6. 常见反模式（禁止）

- 课堂模式只改背景/颜色，不提升信息可读性。
- 把核心读数放在折叠面板深处且无摘要替代。
- 让教师必须多次点击才能看到当前结论。
- 新场景不提供 `presentationSignals` 与 `coreSummary`。
