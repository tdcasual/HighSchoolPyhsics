# 参考页面复刻改造规范

本规范用于把外部现成页面（离线包/HTML）快速改造成本项目场景，实现“逻辑对齐 + 3D复刻 + 项目化控制台”。

## 1. 目标与边界

1. 保留原实验的物理逻辑与状态机行为。
2. 3D 几何与关键参数优先按原页复刻（导线、线圈、仪表、箭头等）。
3. 控制台布局、课堂模式契约、主题配色遵循本项目规范，不强制复制原页视觉。
4. 触屏优先与课堂可读性要求必须满足（1080P 折叠可见核心信息）。

## 2. 必须保留的逻辑（非协商）

1. 状态流转必须与原页一致：
   - 按钮可用/禁用时机
   - 运动中是否允许改参数（如磁极）
   - 记录写入时机（例如“运动完成后记录”）
   - 重置行为（例如“未重置前记录持续累加”）
2. 物理映射必须可追溯：
   - 输入条件（磁极/运动方向）到输出结果（原磁场/磁通变化/感应电流/感应磁场）的映射表
   - 显式实现为可测试的 `model.ts` 纯函数
3. 课堂模式契约必须满足：
   - `SceneLayout` 声明 `presentationSignals`
   - 提供 3-5 行 `coreSummary`
   - 关键读数区带 `data-presentation-signal`

## 3. 实施流程（标准步骤）

1. 拆解原页
   - 解包并定位核心脚本（状态变量、按钮事件、动画循环、记录逻辑）
   - 抽取关键常量（位置、尺寸、速度、公式）
2. 模型先行
   - 在 `model.ts` 固化物理映射与文案格式化函数
   - 先补 `model.test.ts`
3. 状态层对齐
   - 在 `useXxxSceneState.ts` 还原原页状态机与时序
   - 确认“记录累加/重置清空/按钮锁定”行为
4. 3D 复刻
   - `Rig3D` 中优先一比一移植几何参数与路径曲线
   - 允许材质与光照做项目化调整，但不改几何关系
5. 控制台项目化
   - 控件命名、信息分组、字号和按钮尺寸遵循项目已有样式体系
   - 配色遵循 day/night 主题，不强行复刻外部站点皮肤
6. 场景接入
   - 更新 `config/demo-scenes.json`
   - 确保路由自动发现与 Playwright 目录一致

## 4. 测试与验收清单

1. 单测
   - `model.test.ts`：映射正确
   - `useXxxSceneState.test.tsx`：状态机和记录行为正确
2. 结构测试
   - 控件存在、关键按钮行为正确、记录累加正确
3. 课堂模式测试
   - 折叠状态可见 `coreSummary`
   - 信号节点可被查询到
4. 回归验证
   - `npm run lint`
   - `npm run build`
   - 必要时补充路由一致性与目录测试

## 5. 视觉策略约定

1. 3D 场景：结构和运动参数优先忠实原页。
2. 控制台：遵循本项目统一风格（按钮层级、表格可读性、day/night 主题）。
3. 不做“像素级皮肤复刻”，只保证教学信息和交互行为等价。

## 6. 后续协作约定

以后收到“现成页面”请求时，默认按本规范执行：

1. 先审原页逻辑并给出对齐点清单。
2. 直接完成改造与测试，不重复询问是否按同类流程处理。
3. 若原页逻辑与课堂模式契约冲突，以课堂模式非协商原则优先，并在结果中标注差异。
