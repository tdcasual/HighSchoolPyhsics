# 旋转切割磁感线统一场景设计

日期：2026-03-10

## 1. 目标

新增一个面向高中课堂的统一场景，用于在同一页中切换两类常见的感应电动势讨论情形：

1. 竖直向上磁场中，单根导体棒绕一端转动；
2. 水平磁场中，矩形线框绕一条边为轴转动，远离转轴的平行边切割磁感线。

该场景的教学目标不是做复杂仪器复刻，而是让老师快速比较“不同旋转结构下感应电动势大小如何变化”，并在课堂模式下始终保证核心结果可见。

## 2. 核心设计原则

本场景遵循仓库既有课堂模式契约：

- `SceneLayout` 必须声明 `presentationSignals`；
- 必须提供 `coreSummary`，并在控制区折叠后仍可读；
- 关键结果卡片必须带 `data-presentation-signal`；
- 交互遵循触屏优先、一致按钮语义、1080P 首屏可读。

本场景建议定位为“结果读数主导型过程场景”，而不是图表场景：

- `presentationSignals={['live-metric', 'interactive-readout']}`；
- `sceneKind = process`；
- `smartPresentation.layout = never` 或 `enter-only`；
- `coreSummary` 固定 4 行，优先展示实验类型、当前视图、当前转角、感应电动势大小。

## 3. 总体方案

场景采用“单路由、双情形切换”的统一结构，而不是拆成两个独立页面。这样更适合课堂讲授：老师可以先建立“旋转切割磁感线”的共性，再切换具体装置对比差异。

推荐路由：`/rotational-emf`

推荐中文标题：`旋转切割磁感线`

右侧视口保持同一种视觉语言：

- 场景内都有磁场箭头阵列；
- 都有明确的转轴；
- 都高亮“主要切割导体”；
- 都支持 `主视图 / 俯视图` 两种课堂视角。

左侧控制区保持统一结构：

1. 实验类型切换；
2. 参数区；
3. 动画控制；
4. 结果卡片。

## 4. 两个实验情形的表现方式

### 4.1 情形 A：旋转导体棒

- 磁场方向：竖直向上；
- 几何：一个支点 + 一根导体棒；
- 运动：导体棒绕支点在水平面转动；
- 课堂重点：同一根导体棒不同位置线速度不同，但统一输出“当前感应电动势大小”。

该模式下，俯视图最重要，因为它能直接展示圆周运动；主视图则从斜前上方观察，兼顾磁场方向与空间结构。

### 4.2 情形 B：旋转矩形线框

- 磁场方向：水平；
- 几何：薄矩形线框，其中一条边固定为转轴；
- 运动：线框绕转轴翻转；
- 课堂重点：远离转轴的平行边是主要切割边，适合讨论“有效切割长度”和“瞬时速度贡献”。

该模式下，主视图应突出线框翻转姿态；俯视图则用于解释切割边与磁场方向的投影关系。

## 5. 参数设计

为保持课堂简洁，参数只保留最小教学集合，默认值全部为 `1`：

- 磁感应强度 `B = 1`；
- 角速度 `ω = 1`；
- 有效长度 `L = 1`。

状态从零角度开始，点击播放后连续转动。

不建议在第一版加入过多高级参数（如线框宽度、精细材质、相位偏移、极性切换等），遵循 YAGNI 原则，仅围绕“感应电动势大小”展开。

## 6. 结果表达

本场景的核心输出是感应电动势大小，因此结果区建议默认显示：

- `实验类型`；
- `当前视图`；
- `当前转角 θ`；
- `感应电动势 |E|`。

为帮助课堂讲解，可追加一条简短提示语：

- 导体棒模式：`转得越快，棒中的感应电动势越大`；
- 线框模式：`离转轴越远的切割边贡献越明显`。

第一版不强制展示极性正负，只展示大小；如果实现成本低，可在内部保留符号结果，但 UI 层以 `|E|` 为主。

## 7. 物理模型建议

实现时应将物理逻辑集中在 `model.ts` 的纯函数中，而不是散落在 React 组件里。建议拆为：

- `deriveRotatingRodEmfMagnitude(...)`
- `deriveRotatingFrameEmfMagnitude(...)`
- `deriveRotationalEmfReadout(state)`

前两个函数分别处理两个实验模型，第三个函数负责把状态统一映射为视图层需要的读数对象。

这样做的好处：

- UI 与公式解耦；
- 单元测试清晰；
- 后续若要微调公式或默认值，不影响场景结构。

## 8. 视图设计

本场景只提供两个视图按钮：

- `主视图`
- `俯视图`

不再增加底视、侧视或自由切换菜单，避免课堂操作复杂化。

技术上不需要额外做 2D 渲染层，直接采用现有项目惯用的相机预设切换：

- `主视图`：斜前方或斜上方课堂观察角度；
- `俯视图`：近似正顶视角，突出平面关系。

这比单独做一个 SVG/Canvas 2D 图更符合仓库现有模式，也更便于维护。

## 9. 代码结构建议

建议新增目录：`src/scenes/rotational-emf/`

建议文件拆分如下：

- `RotationalEmfScene.tsx`：场景入口，组装 `SceneLayout`；
- `RotationalEmfControls.tsx`：左侧控件与结果卡片；
- `RotationalEmfRig3D.tsx`：3D 几何、磁场箭头、转轴与高亮导体；
- `useRotationalEmfSceneState.ts`：统一状态与动画时序；
- `model.ts`：两个实验的纯函数模型与格式化逻辑；
- `rotational-emf.css`：样式；
- `__tests__/...`：结构、课堂模式、模型、视图切换测试。

## 10. 课堂模式接入建议

推荐接入配置：

- `presentationSignals={['live-metric', 'interactive-readout']}`
- `coreSummaryLineCount = 4`
- `sceneKind = 'process'`
- `smartPresentation.layout = 'never'`
- `smartPresentation.focus = false`
- `smartPresentation.stickySummary = false`

推荐 `coreSummary` 四行：

1. `实验类型: 旋转导体棒 / 旋转矩形线框`
2. `当前视图: 主视图 / 俯视图`
3. `当前转角: ...°`
4. `感应电动势: ... V`

## 11. 测试要求

最少需要覆盖：

- `model.test.ts`：两种模型默认值输出稳定、切换情形后结果改变；
- `useRotationalEmfSceneState.test.tsx`：默认值、播放/暂停、重置、实验切换；
- `structure.test.tsx`：控件、视图按钮、结果读数、模式切换；
- `classroomMode.test.tsx`：课堂模式下 `coreSummary` 可见，结果卡片有 `data-presentation-signal`；
- route/catalog 对齐测试通过。

## 12. 验收标准

当以下条件同时满足时，可认为该场景达到第一版目标：

1. 老师可在一个场景里切换两种旋转切割情形；
2. 默认参数全部为 `1`；
3. 能在 `主视图 / 俯视图` 间切换；
4. 结果区能稳定显示当前感应电动势大小；
5. 课堂模式折叠控制区后，核心摘要仍可见；
6. 场景结构、测试、catalog 元数据全部与项目规范一致。
